# 重构 BnRadio 和 BnRadioGroup 组件事件定义

## 问题分析
根据 [vue-code-skill 规范](file:///h:/lingann/proxy-monitor/.trae/skills/vue-code-skill/SKILL.md#L30-33)，组件必须严格区分：
- **双向绑定事件**：`onUpdate:xxxXxx` 格式（如 `onUpdateModelValue`）
- **普通事件**：必须使用 Props 回调模式（如 `onClick`、`onChange`）

**禁止混合使用 emits 模式和 Props 回调模式。**

## 需要重构的组件

### 1. BnRadio 组件
**当前问题**：使用 `emits: ['update:modelValue', 'change', 'focus', 'blur']` 和 `emit()` 调用

**需要修改**：
- [radio.tsx](file:///h:/lingann/proxy-monitor/src/renderer/components/bn-radio/radio.tsx#L29) - 移除 `emits` 定义
- [radio.tsx](file:///h:/lingann/proxy-monitor/src/renderer/components/bn-radio/radio.tsx#L31) - 移除 `emit` 参数
- [radio.tsx](file:///h:/lingann/proxy-monitor/src/renderer/components/bn-radio/radio.tsx#L78) - 修改 composable 调用，移除 emit 传递
- [radio.tsx](file:///h:/lingann/proxy-monitor/src/renderer/components/bn-radio/radio.tsx#L89) - 将 `emit('update:modelValue', checked)` 改为 `props.onUpdateModelValue?.(checked)`
- [radio.tsx](file:///h:/lingann/proxy-monitor/src/renderer/components/bn-radio/radio.tsx#L92) - 修改 `originalHandleChange` 调用方式
- [radio.tsx](file:///h:/lingann/proxy-monitor/src/renderer/components/bn-radio/radio.tsx#L108-114) - 将 `emit` 调用改为 Props 回调

### 2. BnRadioGroup 组件

### 3. Props 文件更新

#### radio.ts Props
- 添加事件回调 Props：
  - `onUpdateModelValue: Function as PropType<(value: boolean) => void>`
  - `onChange: Function as PropType<(event: Event) => void>`
  - `onFocus: Function as PropType<(event: FocusEvent) => void>`
  - `onBlur: Function as PropType<(event: FocusEvent) => void>`

#### radio-group.ts Props
- 添加事件回调 Props：
  - `onUpdateModelValue: Function as PropType<(value: RadioValueType) => void>`
  - `onChange: Function as PropType<(value: RadioValueType) => void>`

### 4. Composables 更新

#### use-radio.ts
- 移除 `emit` 参数依赖
- 修改事件处理逻辑，直接返回处理函数而非调用 emit

#### use-radio-group.ts
- 移除 `emit` 参数依赖
- 修改事件处理逻辑，直接返回处理函数而非调用 emit
