# 修复样式与图表显示问题

分析发现，当前应用出现“无样式”和“图表缺失”的核心原因是 **CSS Modules 配置不匹配**。

在 `monitor.tsx` 组件中，代码使用了驼峰命名法（如 `styles.statCard`、`styles.dashboardGrid`）来引用样式类，但对应的 SCSS 文件 (`monitor.module.scss`) 定义的是短横线命名（如 `.stat-card`、`.dashboard-grid`）。由于 Vite 默认不自动转换类名为驼峰格式，导致样式对象中的属性为 `undefined`，从而使页面元素丢失了类名，进而导致：
1.  **样式丢失**：所有卡片和布局失去样式，呈现为纯文本堆叠。
2.  **图表不显示**：图表容器 `.chart-container` 样式未应用，高度为 0，导致 ECharts 无法渲染。

## 实施计划

### 1. 配置 Vite 支持 CSS Modules 驼峰命名
修改 `vite.config.ts`，启用 `css.modules.localsConvention: 'camelCase'`。这将允许我们在 JS/TS 中使用驼峰命名（`styles.statCard`）来引用短横线定义的 CSS 类（`.stat-card`），同时保留原有的短横线引用方式，确保兼容性。

### 2. 验证图表组件逻辑
虽然主要原因是样式问题，但我会再次检查 `use-monitor-chart.ts` 和 `monitor.tsx`，确保图表初始化逻辑（`onMounted`、`init`）在容器高度恢复后能正常工作。代码审查显示目前的逻辑是正确的（在 `onMounted` 和 `nextTick` 中初始化），修复样式后图表应能自动恢复显示。

### 预期结果
- 监控页面的“进程”、“连接”等统计信息将以卡片网格形式整齐排列。
- “网络流量”区域将恢复高度，ECharts 流量趋势图将正常显示。
- 设置页面的样式（同样使用了驼峰命名）也将得到修复。
