## 修复计划

修复 [use-field-validation.ts:118](file:///h:/lingann/proxy-monitor/src/renderer/shared/validation/use-field-validation.ts#L118) 的类型报错

**问题**：`result.error.errors[0]` 的类型推断不精确，可能导致 TypeScript 报错

**解决方案**：
1. 在第 121 行使用可选链操作符 `?.` 安全访问 `firstError?.message`
2. 保留原有的回退逻辑，确保即使 `firstError` 为 undefined 也能正常工作

**具体修改**：
```typescript
// 第 121 行，将：
validateMessage.value = firstError?.message || resolveMessage(rules[0], 'default')
// 改为：
validateMessage.value = firstError?.message ?? resolveMessage(rules[0], 'default')
```

使用 `??` (nullish coalescing) 替代 `||` 更精确，只在 `null` 或 `undefined` 时使用回退值，而不会在空字符串时触发回退。
