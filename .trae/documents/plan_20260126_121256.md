# 修改方案：统一使用 BnForm 和 BnFormItem

## 问题分析
1. `component-library.tsx` 使用了不存在的文件路径导入 CommonForm/CommonFormItem
2. `bn-form/index.ts` 中存在向后兼容的别名导出，不符合项目规范

## 修改步骤

### 1. 修改 component-library.tsx
**文件**: `h:\lingann\proxy-monitor\src\renderer\views\component-library\component-library.tsx`

**修改内容**:
- 导入语句：从 `CommonForm/CommonFormItem` 改为 `BnForm/BnFormItem`
- 组件使用：将所有 `<CommonForm>` 改为 `<BnForm>`，`<CommonFormItem>` 改为 `<BnFormItem>`

### 2. 删除 bn-form/index.ts 中的向后兼容导出
**文件**: `h:\lingann\proxy-monitor\src\renderer\components\bn-form\index.ts`

**修改内容**:
- 删除第 18-19 行的别名导出：
  ```typescript
  export { BnForm as CommonForm } from './bn-form'
  export { BnFormItem as CommonFormItem } from './sub-components/bn-form-item'
  ```

## 遵循的规范
- Vue 3 TSX + TypeScript 技术栈
- 组件命名使用 PascalCase (BnForm, BnFormItem)
- 统一使用官方组件名，不使用别名
- 保持代码整洁，删除不必要的向后兼容代码

## 影响范围
- 修改 2 个文件
- 确保项目统一使用 BnForm 和 BnFormItem
