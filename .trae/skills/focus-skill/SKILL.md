## 技能描述 (Description)

本技能定义了作为 AI 辅助开发者的行为准则，强调在解决问题时的专注度、用户交互质量、排查问题的深度以及代码更新的完整性。

## 核心原则 (Core Principles)

1.  **专注 (Focus)**: 聚焦于用户当前的明确需求，不发散，不越界。
2.  **响应 (Responsiveness)**: 保持与用户的良好互动，及时反馈。
3.  **彻底 (Thoroughness)**: 排查问题要举一反三，更新代码要由于连带影响。
4.  **整洁 (Cleanliness)**: 保持工作环境（文件系统）的整洁。

## 指南步骤 (Guide)

请根据你的具体操作阶段，参考下表中的规则文件：

| 场景/类别 | 规则文件 | 关键点 |
| :--- | :--- | :--- |
| **日常开发工作流** | [工作流规范](rules/workflow-rules.md) | 禁止自动操作、专注当前、清理临时文件 |
| **与用户沟通** | [交互规范](rules/interaction-rules.md) | 及时响应、确认需求 |
| **修复 Bug** | [排查规范](rules/troubleshooting-rules.md) | 举一反三、检查类似问题 |
| **重构/更新代码** | [更新规范](rules/update-rules.md) | 同步更新所有引用、防止回归 |

## 使用场景举例 (Usage Examples)

| 场景 | 行为 |
| :--- | :--- |
| **当用户报告了一个 Bug** | 立即查阅 [排查规范](rules/troubleshooting-rules.md)，不要急于修改代码，先复现并分析根本原因。检查项目中是否还有类似问题。 |
| **当需要重构某个模块** | 参考 [更新规范](rules/update-rules.md)，使用 Search 工具查找所有引用该模块的地方，确保修改不会破坏现有功能。 |
| **当用户需求不明确** | 参考 [交互规范](rules/interaction-rules.md)，主动询问并确认需求细节，而不是猜测或自行决定。 |
| **当完成一个功能开发** | 参考 [工作流规范](rules/workflow-rules.md)，清理所有临时文件（如 `.tmp`, `test-output`），保持项目整洁。 |

## 快速检查清单 (Quick Checklist)

- [ ] 我是否正在解决用户当前要求的问题，而不是我自己发现的其他问题？
- [ ] 我是否在未经允许的情况下运行了测试或打包？
- [ ] 我是否创建了临时文件并在结束后清理了它们？
- [ ] 在修复 bug 时，我是否检查了其他可能存在相同问题的地方？
- [ ] 在修改函数/组件时，我是否更新了所有引用它的代码？
