# 工作流规范 (Workflow Rules)

本文件定义了开发过程中的工作流规范，旨在提高效率、保持专注并维护项目整洁。

## 规则列表

### 1. 禁止自动测试/打包
**规则**: 在用户没有特别要求的情况下，禁止进行自动测试运行、打包操作。
**指南**: 这些操作通常耗时较长，除非用户明确要求验证或发布，否则不要主动执行。

### 2. 专注当前问题
**规则**: 禁止过度考虑，始终专注于解决用户当前遇到的问题，而不是解决其他潜在问题。
**指南**: “不要在修自行车的时候去修屋顶”。如果发现了其他问题，可以记录下来或告知用户，但不要偏离当前任务。

### 3. 禁止需求蔓延
**规则**: 禁止需求蔓延，保持用户需求的聚焦和优先级。
**指南**: 如果用户在对话中提出了新的、无关的需求，礼貌地建议在当前任务完成后再处理。

### 4. 及时清理
**规则**: 任何临时的文件或目录，都要在使用完毕后及时删除，避免干扰项目现有文件，保持项目目录的整洁和有序。
**指南**: 创建测试文件、临时脚本或解压的资源后，务必进行清理。

**示例**:
```bash
# Bad
# 留下了 temp.txt

# Good
# 使用完毕后
rm temp.txt
```
