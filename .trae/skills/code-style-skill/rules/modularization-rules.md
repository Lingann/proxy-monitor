# 模块化规范 (Modularization Rules)

本文件包含了代码模块化的设计规范，旨在提高代码的复用性、可维护性和低耦合度。

## 规则列表

### 1. 单一职责
**规则**: 避免在一个文件中定义多个函数或类，保持每个文件的职责单一。
**指南**: 一个文件应该只包含一个主要的类、组件或相关的一组紧密耦合的函数。

**示例**:
**Bad**: `utils.ts` 包含了日期处理、HTTP 请求、验证逻辑等。
**Good**: 拆分为 `date-utils.ts`, `http-client.ts`, `validator.ts`。

### 2. 逻辑清晰
**规则**: 每个函数或类的功能要保持清晰和可理解，避免复杂的逻辑。
**指南**: 函数应该短小精悍，只做一件事。

### 3. 清理废弃模块
**规则**: 及时清理不再需要的模块，避免引入不必要的依赖。
**指南**: 定期审查代码库，删除未使用的文件和导出。

### 4. 共享通用模块
**规则**: 比较通用的功能模块，可以被抽离出来共享给其他项目使用的，可以放到一个独立的目录中。
**指南**: 识别核心业务逻辑与通用工具逻辑，将通用部分解耦。
