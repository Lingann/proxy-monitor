# 注释规范 (Comment Rules)

本文件包含了代码注释的编写规范，旨在提高代码的可理解性和文档质量。

## 规则列表

### 1. 注释格式
**规则**: 注释优先使用 `/* ... */` 格式，且注释内容要中文，并且注释尽可能独占一行而非多行。
**指南**: 即使是单行注释，也推荐使用 `/* ... */` 风格，以保持统一。

**示例**:
```typescript
// Bad
// 这是一个变量
const count = 0;

// Good
/* 这是一个变量 */
const count = 0;
```

### 2. 注释语言
**规则**: 注释全部用中文。
**指南**: 确保所有团队成员都能无障碍阅读，描述要准确、通顺。

### 3. 广泛添加注释
**规则**: 在尽可能多的地方添加单行 `/* ... */` 格式的注释，例如：函数、类、变量、常量等。
**指南**: 不要假设代码是自解释的。对于复杂的逻辑、关键的变量、导出的接口，必须添加注释。

**示例**:
```typescript
/* 用户服务类 */
class UserService {

  /* 获取用户信息的 API 地址 */
  private readonly apiUrl = '/api/users';

  /* 根据 ID 获取用户 */
  async getUser(id: string): Promise<User> {
    // ...
  }
}
```
