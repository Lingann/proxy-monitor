## 命令

1. 禁止在代码中过渡使用嵌套，保持代码的可读性和简洁性
2. 优先使用卫语句（Guard Clauses）来处理边界情况，避免使用嵌套的 if 语句。
3. 使用提前返回（Early Returns）来处理简单的情况，避免使用复杂的嵌套结构。
4. 避免在循环中使用复杂的逻辑，保持循环的简洁性。
5. 及时清理不再需要的变量和资源，避免内存泄漏。
6. 不用考虑向后兼容，保持代码的简洁性和可维护性。
7. 注释优先使用 /* ... */ 格式，且注释内容要中文， 并且注释尽可能独占一行而非多行。
8. 禁止未import 就使用的变量或函数，必须先import 后使用，并且注意检查。
9. 单行语句优先于多行语句，保持代码的简洁性。例如判断语句、赋值语句等。
10. 能不用try-catch 就不要用，保持代码的简洁性。
11. 禁止在赋值前使用未定义的变量或函数，必须先定义后使用。
12. 禁止使用any 类型，必须使用具体类型或泛型。
13. 注释全部用中文
14. 在尽可能多的地方添加单行/* ... */ 格式的注释， 例如：函数、类、变量、常量等。
15. 一行代码都加一行空格，尤其调用函数、重要的执行、判断、变量定义等。

模块化规则：
1. 避免在一个文件中定义多个函数或类，保持每个文件的职责单一。
2. 每个函数或类的功能要保持清晰和可理解，避免复杂的逻辑。
3. 及时清理不再需要的模块，避免引入不必要的依赖。
4. 禁止重复导出，比如在 index.ts 中导出了某个模块，那么在其他模块中就不应该再导出该模块。
5. 每个目录模块下，都必须有存在一个 index.ts 文件，用于导出该目录下的所有模块。
6. 比较通用的功能模块，可以被抽离出来共享给其他项目使用的，可以放到一个独立的目录中。
7. 禁止使用CommonJS 模块， 项目只能使用ES6 模块规范。

## 使用场景

1. 编写任何 TypeScript/JavaScript 代码时
2. 进行代码重构或优化时
3. 创建新的模块或文件时

## 解释

1. 保持代码简洁、可读、易维护是首要目标
2. 模块化有助于代码复用和隔离
